# Scaling African Language Datasets

Africa has over 2,000 languages. Most of these languages remain low-resource. Local communities work hard to create new data, but these efforts often stop too soon. We need open collaboration to make sure research lasts. Open data lets datasets grow as language changes over time.

### The Challenge of Data Curation

It is hard to build datasets. Manual work by humans costs a lot of money, and it is slow. Computers can create synthetic data very fast. However, computer-made data has problems. It can be biased. It often misses the way native speakers actually talk. We need a way to get the scale of AI with the quality of humans.

### Our Three Step Process

We use a "human-in-the-loop" model. This reduces manual labor but keeps quality high.

1. **Source.** We find high-quality data that already exists.
2. **Translate.** We use AI models to create initial translations.
3. **Validate.** Native speakers and linguists check every sentence for accuracy.

### Our Impact in Practice

We have already used this method to expand important datasets.

- **The Mbogho Dataset.** We added a Kiswahili-English component to a set of 29,280 sentences. This helps computers move knowledge between languages more easily.
- **AfriXNLI.** This project started with 1,050 samples per language. We grew this to 5,000 samples for several languages. We also added entirely new languages that were not there before.

Our approach shows that human and AI collaboration works. It provides a steady way to build resources for African languages. We will keep looking for better ways to curate data.
